[FACT] TOST EV/Sales 2.31 vs XYZ(Block) 1.50 vs LSPD 0.62.

## Phase 5: Cross-Layer — Bottleneck Hunter + §0.9 AI Filter

### 핵심 질문 답변

**1. "TOST의 해자가 AI로 대체되는가?"**
→ **아니오.** 물리적 POS 하드웨어 설치 + 레스토랑 직원 교육 + 시스템 통합은 AI가 대체 불가. AI는 오히려 Toast 플랫폼 **위에서** 구동되는 도구. Toast = AI의 OS, AI = Toast의 기능.

**2. "AI가 발전할수록 TOST가 더 가치 있어지는가?"**
→ **그렇다 [THESIS].** 12만+ 레스토랑의 거래/메뉴/고객/재고 데이터 = AI 학습의 원료. AI가 잘 작동할수록 → Toast의 ARPU 상승 → 고객 lock-in 강화.

**3. "데이터 축적형인가 실시간형인가?"**
→ **축적형.** 매일 수백만 거래, 메뉴 변경, 고객 행동 패턴이 누적. 시간이 갈수록 데이터 우위 확대.

**4. "전환비용은?"**
→ **매우 높음.** POS 교체 = 하드웨어 교체 + 소프트웨어 마이그레이션 + 직원 재교육 + 운영 중단 리스크. 일반적으로 3-5년 lock-in.

**5. "12만 → 100만: 10x 가능한가?"**
→ [THESIS] TAM 100만 레스토랑 중 디지털화 의향 40-50만으로 축소하면 → 12만/45만 = 27% 침투율. 나머지 73%를 가져가면 3.7x. 여기에 ARPU 2x 성장(AI 제품) 가하면 **7-8x revenue 가능**. 10x는 국제 확장 필요.

### Cross-Layer Bottleneck Assessment
- **현재 병목:** 미국 레스토랑 SMB의 디지털 전환 속도 (영업 인력 한계)
- **차기 병목:** 데이터 활용/AI 수익화 능력
- **병목 소유자:** Toast (12만 설치 기반 = 데이터 독점)
