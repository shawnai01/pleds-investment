## Phase 3: Layer 3 — Value Chain Analysis

### Causal Mechanism Map: NVIDIA

```
[AI 모델 학습/추론 수요] 
  → [GPU 설계 (NVDA: CUDA + 아키텍처)] 
    → [파운드리 제조 (TSMC)] 
      → [HBM 메모리 (SK Hynix/Samsung)] 
        → [서버 조립 (SMCI, Dell)] 
          → [데이터센터 배치 (MSFT, GOOGL, META, AMZN)]
            → [AI 서비스/애플리케이션 (OpenAI, Anthropic 등)]

핵심 노드: GPU 설계 + CUDA 에코시스템
치명적 약점:
1. TSMC 단일 파운드리 의존 (대만 리스크)
2. Custom ASIC이 inference 시장 잠식 (Google TPU, Amazon Trainium)
3. CUDA lock-in 깨지는 경우 (OpenAI + AMD MI450 deal)

대안 경로:
- AMD ROCm → CUDA 대안, 그러나 에코시스템 격차 여전히 큼
- Custom ASIC → training에서는 아직 비현실적, inference에서 점진적 침투
- Intel Gaudi → 가격 경쟁력만, 성능 열위
```

### AI Commoditization Filter 적용 🤖

**Q1: NVDA의 핵심 가치가 AI로 대체/commoditize되는가?**
- NVDA는 AI를 만드는 도구를 만든다 → AI가 GPU를 대체하지 않음
- 🟢 **물리적 인프라/설계 IP** — AI가 칩 설계를 더 잘 하게 될 수 있으나, 제조/에코시스템은 대체 불가

**Q2 (Search Problem Lens): NVDA의 핵심 문제가 compute + search로 풀려버리는가?**
- 칩 설계 일부는 AI로 가속 가능 (NVDA 자체가 이를 활용 중)
- 그러나 CUDA 에코시스템(4M+ 개발자, 수천 라이브러리)은 네트워크 효과 — search로 치환 불가
- 🟡 **부분 치환** — 설계 프로세스는 가속되나, 에코시스템 해자는 강화

**Q3 (Verification Moat): NVDA만이 생성할 수 있는 보상 신호가 있는가?**
- CUDA 에코시스템의 피드백 루프: 개발자가 CUDA로 개발 → 더 많은 라이브러리 → 더 많은 개발자
- 🟢 **네트워크 효과 + 데이터 축적** — 쓸수록 해자 확대
- 그러나 **물리적 Atom World 해자는 약함** — NVDA는 fabless, 물리적 자산은 TSMC 소유

**종합 판정:** 🟢 Strong (CUDA 네트워크 효과) + 🟡 Partial Risk (fabless 구조, custom ASIC 침투)

### Round 2 — Moat Breaker 반론

**"CUDA lock-in이 깨지는 시나리오를 과소평가하고 있다."**
1. OpenAI가 AMD MI450과 대규모 계약 → "킬러 앱"이 CUDA 밖에서 작동하면 개발자 이동 시작

4. GPU 시장 점유율 전망: 95/5 → 70/30 (analyst estimate) [THESIS]

### Round 3 — 재반박/수정

**Moat Analyst → defend (부분 수정):**
1. OpenAI+AMD: 아직 실제 대규모 training 결과 없음. CUDA 에코시스템 전환 비용은 수조 달러 규모. **defend.**

3. Google TPU: Google은 특수 사례 — 자체 모델 최적화. 범용 시장과 다름. **defend.**
