> 적용 대상: METHODOLOGY.md, WORKFLOW.md, EXPERTS.md, PORTFOLIO.md, DATA-SOURCES.md

## 감사 5축 (Five Audit Axes)

### A1. 일관성 (Consistency) 🔗
> "5개 문서가 하나의 시스템으로 작동하는가?"

| # | 점검 항목 | 기준 |
|---|----------|------|
| A1.1 | **용어 통일** | 같은 개념에 같은 이름. 전문가 이름, Phase 번호, 역할 설명이 문서 간 동일 |
| A1.2 | **숫자 정합** | 전문가 수, Layer 수, Loop 횟수, 가중치 등 정량 값이 문서 간 일치 |
| A1.3 | **절차 정합** | WORKFLOW의 Phase 순서 ↔ METHODOLOGY의 절차 설명 ↔ EXPERTS의 역할 범위가 모순 없음 |
| A1.4 | **상호 참조 유효** | "§10 참조", "METHODOLOGY §3-1 연동" 등 교차 참조가 실제로 유효 |
| A1.5 | **버전 동기화** | 5개 문서의 버전 번호가 동일 에포크 기준 (예: 모두 v4.1) |

**판정:** 불일치 0개 = ✅ | 1-2개 = 🟡 | 3개+ = 🔴

### A2. 필요성 (Necessity) — "바닐라 원칙" 🍦
> "이 모듈/절차가 없으면 분석 결과가 달라지는가?"

| # | 점검 항목 | 기준 |
|---|----------|------|
| A2.1 | **모듈 정당화** | 각 모듈(§ 섹션)이 최근 3개월 내 판정을 실제로 변경한 사례 존재 |
| A2.2 | **전문가 정당화** | 각 전문가가 최근 분석에서 고유한 기여(다른 전문가가 제시하지 못한 관점) 존재 |
| A2.3 | **절차 정당화** | 각 절차 단계가 삭제 시 명확히 품질 저하 초래 |
| A2.4 | **복잡도 예산** | 총 모듈 수, 전문가 수, 최대 debate round 수가 기준선 대비 증가하지 않았는지 |
| A2.5 | **중복 검출** | 2개 이상 문서에서 같은 내용을 다른 표현으로 설명하고 있지 않은지 |

**바닐라 테스트 (모듈별):**
```
Q: "이 모듈을 삭제하면 최근 3건의 분석 결과가 달라졌을까?"
YES → 유지
NO  → 삭제/통합 후보
```

**판정:** 불필요 모듈 0개 = ✅ | 1-2개 = 🟡 | 3개+ = 🔴

### A3. 직교성 (Orthogonality) 🔀
> "각 전문가/모듈이 독립적이고 겹치지 않는 관점을 제공하는가?"

| # | 점검 항목 | 기준 |
|---|----------|------|
| A3.1 | **전문가 직교성** | 전문가 A의 output을 전문가 B가 대체 불가능 |
| A3.2 | **Layer 직교성** | Layer 간 input/output이 명확히 구분됨 (L1 output = L2 input) |
| A3.3 | **역할 분리** | Moderator/Critic/Analyst 3자 역할이 혼합되지 않음 |
| A3.4 | **관점 커버리지** | 투자 판단에 필요한 핵심 관점이 누락 없이 커버 |
| A3.5 | **통합 전문가 내적 일관성** | 2개 모델을 통합한 전문가(예: Dalio+Druckenmiller)가 내적 충돌 없이 운용 가능 |

**직교성 매트릭스 테스트:**
```
모든 전문가 쌍 (A, B)에 대해:
Q: "A의 분석을 읽은 후 B의 분석이 새로운 정보를 추가하는가?"
YES → 직교 ✅
NO  → 중복 → 통합 후보 🔴
```

**판정:** 중복 쌍 0개 = ✅ | 1개 = 🟡 | 2개+ = 🔴

### A4. 복잡도 적정성 (Complexity Adequacy) ⚖️
> "필요한 만큼 복잡하고, 그 이상은 아닌가?"

| # | 점검 항목 | 기준 |
|---|----------|------|
| A4.1 | **실행 가능성** | 기술된 절차가 현재 인프라(API, 모델, 시간)로 실제 실행 가능 |
| A4.2 | **토큰 예산 현실성** | Standard Day ~30K, Event Day ~80-100K 추정이 실제와 부합 |
| A4.3 | **시간 예산 현실성** | 일일 브리핑 09:00 KST 내 완료 가능 |
| A4.4 | **비용 예산 현실성** | 일일/주간/월간 비용이 지속 가능 |
| A4.5 | **인지 부하** | Shawn이 산출물을 읽고 행동으로 변환하는 데 걸리는 시간 적정 |

**복잡도 스모크 테스트:**
```
Q1: "새로 합류한 분석가가 이 5개 문서를 읽고 1시간 내에 시스템을 이해할 수 있는가?"
Q2: "가장 단순한 분석(Standard Day)이 15분 내에 완료되는가?"
Q3: "가장 복잡한 분석(Deep Loop 5회)이 2시간 내에 완료되는가?"
```

**판정:** 3개 YES = ✅ | 1-2개 NO = 🟡 | 3개 NO = 🔴

### A5. 철학 정렬 (Philosophy Alignment) 🚀🔩
> "Rocket with Tight Bolts를 실현하는 conviction bet 파트너인가?"

| # | 점검 항목 | 기준 |
|---|----------|------|
| A5.1 | **Dual Thinking 실현** | Alpha Thinking과 Fact Grounding이 실제로 분리 운용됨 |
| A5.2 | **Edge 생성 능력** | 시스템이 컨센서스와 다른 독자적 관점을 실제로 생성 |
| A5.3 | **Anti-Echo Chamber** | Bull/Bear 균형, Critic 실효성, 관성 차단 메커니즘 작동 |
| A5.4 | **Action Orientation** | 산출물이 "좋은 분석"이 아니라 "실행 가능한 투자 결정"으로 수렴 |
| A5.5 | **ICL 실효성** | Shawn의 실제 상황(시드, 목표, 리스크 성향)이 모든 제안에 반영 |
| A5.6 | **자기 교정** | PT 시스템, 적중률 추적, 피드백 루프가 실제 작동 |

**철학 정렬 최종 질문:**
```
"이 시스템이 Shawn에게 '분석 보고서'를 주는가, '투자 파트너의 판단'을 주는가?"
```

**판정:** 보고서 → 🔴 재설계 | 파트너 판단 → ✅

## 감사 실행 절차

### Step 1: 독립 읽기 (30분)
- 5개 문서를 **처음 읽는 사람의 관점**에서 통독
- 각 문서 읽는 동안 다른 문서 참조 금지 (교차 참조 유효성 나중에 검증)
- 의문점, 혼란, 중복, 모순 즉시 기록

### Step 2: 교차 대조 (20분)
- A1 일관성 축 점검: 용어, 숫자, 절차, 참조, 버전
- grep 기반 키워드 검색으로 정량 교차 검증

### Step 3: 바닐라 테스트 (15분)
- A2 필요성 축: 각 모듈/전문가에 "삭제 시 결과 변경?" 질문
- A4 복잡도 축: 스모크 테스트 3개 실행

### Step 4: 직교성 매트릭스 (15분)
- A3 직교성 축: 전문가 쌍별 중복 검사
- 커버리지 갭 검사

### Step 5: 철학 정렬 (10분)
- A5 철학 축: 최종 질문으로 전체 시스템 평가

### Step 6: 보고서 작성
- 축별 판정 + 구체적 발견 + 수정 권고
- 등급: A(우수) / B(양호) / C(개선 필요) / D(재설계 필요)
- 산출물: `audit/YYYY-QQ-constitution-audit.md`
